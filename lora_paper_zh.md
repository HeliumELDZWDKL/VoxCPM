## 摘要

自然语言处理中一个重要的范式是：先在通用领域数据上进行大规模预训练，再针对特定任务或领域进行适配。随着预训练模型越来越大，重新训练所有模型参数的全量微调（full fine-tuning）变得越来越不可行。以 GPT-3 175B 为例——为每个任务部署一个独立的微调模型实例，每个都有 1750 亿参数，成本高得令人望而却步。

我们提出了 **低秩自适应（Low-Rank Adaptation，LoRA）** 方法：冻结预训练模型的权重，在 Transformer 架构的每一层注入可训练的秩分解矩阵，从而大幅减少下游任务的可训练参数量。与使用 Adam 优化器全量微调的 GPT-3 175B 相比，LoRA 可以将可训练参数量减少 **10,000 倍**，GPU 显存需求减少 **3 倍**。在 RoBERTa、DeBERTa、GPT-2 和 GPT-3 上，LoRA 在模型质量上与全量微调持平甚至更优，同时拥有更少的可训练参数、更高的训练吞吐量，并且与 Adapter 方法不同，LoRA **不引入额外的推理延迟**。我们还对语言模型适配中的秩缺陷进行了实证研究，揭示了 LoRA 有效性的内在机理。我们在 https://github.com/microsoft/LoRA 发布了便于将 LoRA 集成到 PyTorch 模型中的工具包，以及 RoBERTa、DeBERTa 和 GPT-2 的实现和模型权重。

## 1 引言

自然语言处理中的许多应用依赖于将**一个**大规模预训练语言模型适配到**多个**下游应用。这种适配通常通过微调（fine-tuning）来完成，即更新预训练模型的所有参数。微调的主要缺点在于：新模型的参数量与原始模型一样多。随着更大的模型每隔几个月就会被训练出来，这从 GPT-2 或 RoBERTa large 的"小麻烦"变成了 GPT-3（1750 亿可训练参数）的**严重部署挑战**。

> 注：虽然 GPT-3 175B 通过少样本学习（few-shot learning）就能取得不错的表现，但微调能显著提升其性能（见附录 A）。

许多研究试图通过只适配部分参数或为新任务学习外部模块来缓解这一问题。这样，对于每个任务，我们只需在预训练模型之外存储和加载少量任务专用参数，极大地提升了部署时的运行效率。然而，现有技术往往通过增加模型深度来**引入推理延迟**（Houlsby 等, 2019; Rebuffi 等, 2017），或者**减少模型可用的序列长度**（Li & Liang, 2021; Lester 等, 2021）（见第 3 节）。更重要的是，这些方法通常无法达到全量微调的基准水平，在效率和模型质量之间存在权衡。

我们受到 Li 等(2018a) 和 Aghajanyan 等(2020) 的启发——他们表明，学到的过参数化模型实际上位于一个低内在维度上。我们假设，模型适配过程中权重的变化也具有低"内在秩"，由此提出了我们的 **低秩自适应（LoRA）** 方法。LoRA 允许我们通过优化适配过程中稠密层变化量的秩分解矩阵来间接训练神经网络中的某些稠密层，同时保持预训练权重冻结，如图 1 所示。以 GPT-3 175B 为例，我们展示了即使全秩（即 d）高达 12,288，一个非常低的秩（即图 1 中的 r 可以是 1 或 2）就已足够，使 LoRA 在存储和计算上都非常高效。

**LoRA 具有以下几个关键优势：**

- 一个预训练模型可以被共享，并用于为不同任务构建许多小的 LoRA 模块。我们可以冻结共享模型，通过替换图 1 中的矩阵 A 和 B 来高效切换任务，显著减少存储需求和任务切换开销。

- LoRA 使训练更高效，并将使用自适应优化器时的硬件门槛降低最多 3 倍，因为我们不需要为大多数参数计算梯度或维护优化器状态。我们只需优化注入的、小得多的低秩矩阵。

- 我们简洁的线性设计允许在部署时将可训练矩阵与冻结的权重合并，从构造上保证与全量微调模型相比**不引入任何推理延迟**。

- LoRA 与许多现有方法正交，可以与它们组合使用，例如前缀调优（prefix-tuning）。我们在附录 E 中提供了一个示例。

## 2 问题描述

虽然我们的方法不限于特定的训练目标，但我们以语言建模作为主要应用场景。以下是语言建模问题的简要描述，特别是在给定特定任务提示下的条件概率最大化。

假设我们有一个预训练的自回归语言模型 P_Φ(y|x)，由参数 Φ 参数化。例如，P_Φ(y|x) 可以是基于 Transformer 架构的通用多任务学习器，如 GPT。考虑将此预训练模型适配到下游的条件文本生成任务，如摘要生成、机器阅读理解（MRC）和自然语言转 SQL（NL2SQL）。每个下游任务由一个上下文-目标对的训练数据集表示：Z = {(x_i, y_i)}，其中 x_i 和 y_i 都是 token 序列。

在全量微调中，模型被初始化为预训练权重 Φ_0，并通过反复沿梯度方向更新为 Φ_0 + ΔΦ，以最大化条件语言建模目标：

```
max_Φ Σ_(x,y)∈Z Σ_t log(P_Φ(y_t | x, y_<t))
```

全量微调的主要缺点是，对于**每个**下游任务，我们需要学习一个**不同的**参数增量 ΔΦ，其维度 |ΔΦ| 等于 |Φ_0|。因此，如果预训练模型很大（例如拥有约 1750 亿参数的 GPT-3），存储和部署许多独立的微调模型实例是极具挑战性的。

在本文中，我们采用了一种更参数高效的方法，其中任务专用的参数增量 ΔΦ = ΔΦ(Θ) 由一个小得多的参数集 Θ 进一步编码，其中 |Θ| ≪ |Φ_0|。寻找 ΔΦ 的任务因此变成了对 Θ 的优化。在后续章节中，我们提出使用低秩表示来编码 ΔΦ，这在计算和存储方面都很高效。**当预训练模型是 GPT-3 175B 时，可训练参数 |Θ| 可以小到 |Φ_0| 的 0.01%。**

## 3 现有方案不够好吗？

我们要解决的问题并不新鲜。自迁移学习诞生以来，已有数十项工作致力于使模型适配更加参数和计算高效。以语言建模为例，在高效适配方面有两种主流策略：**添加 Adapter 层** 或 **优化某些形式的输入层激活**。然而，两种策略都有其局限性，尤其是在大规模和对延迟敏感的生产场景中。

### Adapter 层引入推理延迟

Adapter 有许多变体。虽然可以通过剪枝层或利用多任务设置来减少整体延迟，但没有直接的方法来绕过 Adapter 层中的额外计算。这看似不是问题，因为 Adapter 层被设计为参数量很少（有时不到原始模型的 1%）。然而，大型神经网络依赖硬件并行性来保持低延迟，而 Adapter 层必须**顺序处理**。这在在线推理场景中影响很大，因为此时批处理大小通常只有 1。

在没有模型并行的通用场景中，例如在单个 GPU 上运行 GPT-2 medium 的推理，即使 Adapter 的瓶颈维度非常小，我们也能看到**明显的延迟增加**（见表 1）。

> 表 1 结果：在序列长度 128、批大小 1 的场景下，Adapter_H 带来了 **30.3%** 的延迟增加。

### 直接优化 Prompt 很困难

另一个方向，如前缀调优（prefix tuning）所示，面临不同的挑战。我们观察到前缀调优**难以优化**，且其性能随可训练参数的变化呈非单调变化。更根本的是，为适配保留一部分序列长度必然会减少可用于处理下游任务的序列长度。

## 4 我们的方法

我们描述 LoRA 的简洁设计及其实际优势。这里阐述的原则适用于深度学习模型中的任何稠密层，尽管我们在实验中只关注 Transformer 语言模型中的某些权重。

### 4.1 低秩参数化更新矩阵

神经网络包含许多执行矩阵乘法的稠密层。这些层中的权重矩阵通常是满秩的。Aghajanyan 等(2020) 表明，预训练语言模型具有低"内在维度"，即使随机投影到较小的子空间也能高效学习。受此启发，我们假设权重在适配过程中的更新也具有低"内在秩"。

对于预训练权重矩阵 W_0 ∈ R^(d×k)，我们通过低秩分解来约束其更新：

```
W_0 + ΔW = W_0 + BA
```

其中 B ∈ R^(d×r)，A ∈ R^(r×k)，且秩 r ≪ min(d, k)。

**训练时**，W_0 被冻结且不接收梯度更新，而 A 和 B 包含可训练参数。W_0 和 ΔW = BA 乘以相同的输入，它们各自的输出向量按坐标相加。对于 h = W_0·x，修改后的前向传播为：

```
h = W_0·x + ΔW·x = W_0·x + B·A·x
```

我们使用**随机高斯分布初始化 A**，**用零初始化 B**，因此训练开始时 ΔW = BA 为零。然后我们用 α/r 来缩放 ΔW·x，其中 α 是一个关于 r 的常数。当用 Adam 优化时，调整 α 大致等同于调整学习率。因此，我们简单地**将 α 设为第一次尝试的 r**，不再调优。这种缩放有助于减少我们改变 r 时重新调优超参数的需要。

**全量微调的推广：** 更一般的微调形式允许训练预训练参数的子集。LoRA 更进一步，不要求权重矩阵在适配过程中的累积梯度更新具有满秩。这意味着，当对所有权重矩阵应用 LoRA 并训练所有偏置时，通过将 LoRA 秩 r 设为预训练权重矩阵的秩，我们大致恢复了全量微调的表达能力。

**无额外推理延迟：** 部署时，我们可以显式计算并存储 W = W_0 + BA，然后像平常一样进行推理。当需要切换到另一个下游任务时，我们可以减去 BA 恢复 W_0，然后加上不同的 B'A'，这是一个具有很小内存开销的快速操作。**关键是，这从构造上保证了我们不会引入任何额外的推理延迟。**

### 4.2 将 LoRA 应用于 Transformer

原则上，我们可以将 LoRA 应用于神经网络中任意权重矩阵的子集。在 Transformer 架构中，自注意力模块有四个权重矩阵（W_q, W_k, W_v, W_o），MLP 模块有两个。我们将研究限制为**仅适配注意力权重**，冻结 MLP 模块，这出于简洁性和参数效率的考虑。我们在第 7.1 节进一步研究了适配不同类型注意力权重矩阵的效果。

**实际优势和局限：** 最显著的好处来自存储和显存使用的减少。对于用 Adam 训练的大型 Transformer，如果 r ≪ d_model，我们可以将显存使用减少最多 **2/3**，因为我们不需要为冻结的参数存储优化器状态。在 GPT-3 175B 上，我们将训练期间的显存消耗从 **1.2TB 降至 350GB**。当 r=4 且仅适配 query 和 value 投影矩阵时，检查点大小减少了约 **10,000 倍**（从 350GB 降至 35MB）。

## 5 实验

我们在 RoBERTa、DeBERTa、GPT-2 上评估 LoRA 的下游任务性能，然后扩展到 GPT-3 175B。实验涵盖从自然语言理解（NLU）到生成（NLG）的广泛任务。

### 5.1 基准方法

- **全量微调（FT）**：更新所有模型参数
- **Bias-only / BitFit**：仅训练偏置向量
- **前缀嵌入调优（PreEmbed）**：在输入 token 中插入特殊的可训练 token
- **前缀层调优（PreLayer）**：在每个 Transformer 层之后替换激活值
- **Adapter 调优**：在自注意力和 MLP 模块之间插入适配器层
- **LoRA**：在现有权重矩阵旁添加可训练的秩分解矩阵对

### 5.2 RoBERTa base/large 实验结果

在 GLUE 基准测试上：
- **RoBERTa base + LoRA**（仅 0.3M 可训练参数）：平均 87.2%
- **RoBERTa base + 全量微调**（125M 参数）：平均 86.4%
- **LoRA 以 0.3M 参数超越了 125M 参数的全量微调！**

### 5.3 DeBERTa XXL 实验结果

- **DeBERTa XXL + 全量微调**（1500M 参数）：GLUE 平均 91.1%
- **DeBERTa XXL + LoRA**（仅 4.7M 参数）：GLUE 平均 **91.3%**
- **LoRA 参数量仅为全量微调的 0.3%，性能却略优！**

### 5.4 GPT-2 Medium/Large 实验结果

在 E2E NLG Challenge 上：
- **GPT-2 Medium + LoRA**（0.35M 参数）：BLEU 70.4
- **GPT-2 Medium + 全量微调**（354.92M 参数）：BLEU 68.2
- **LoRA 以千分之一的参数量超越全量微调！**

### 5.5 扩展到 GPT-3 175B

作为最终压力测试，在 GPT-3 175B 上的结果：

| 方法 | 可训练参数 | WikiSQL | MNLI-m | SAMSum |
|------|-----------|---------|--------|--------|
| GPT-3 全量微调 | 175,255.8M | 73.8 | 89.5 | 52.0/28.0/44.5 |
| GPT-3 + LoRA | 4.7M | **73.4** | **91.7** | **53.8/29.8/45.9** |

**LoRA 以仅 4.7M 参数（占 0.003%）在 MNLI 和 SAMSum 上超越了全量微调！**

## 6 相关工作

- **Transformer 语言模型**：自 BERT 和 GPT-2 以来，在通用数据上预训练后在任务专用数据上微调已成为主流范式。GPT-3 是迄今为止最大的单 Transformer 模型（1750 亿参数）。

- **提示工程与微调**：GPT-3 175B 可以通过少量示例来适配行为，但结果严重依赖输入提示。微调能显著提升性能，但 GPT-3 的巨大规模使得常规微调面临挑战。

- **参数高效适配**：许多方法在现有层之间插入 Adapter 层。LoRA 使用类似的瓶颈结构来施加低秩约束，但关键区别在于——LoRA 学到的权重可以在推理时与主权重合并，从而**不引入延迟**。

- **深度学习中的低秩结构**：低秩结构在机器学习中非常常见。据我们所知，没有先前的工作考虑过对冻结模型进行低秩更新以适配下游任务。

## 7 理解低秩更新

### 7.1 应该对 Transformer 中的哪些权重矩阵应用 LoRA？

在 GPT-3 175B 上，给定 18M 参数预算的实验结果：

| 权重类型 | WikiSQL | MultiNLI |
|---------|---------|----------|
| 仅 W_q | 70.4 | 91.0 |
| 仅 W_k | 70.0 | 90.8 |
| 仅 W_v | 73.0 | 91.0 |
| 仅 W_o | 73.2 | 91.0 |
| **W_q + W_v** | **73.7** | **91.3** |
| W_q + W_k + W_v + W_o | 73.7 | 91.7 |

**结论：同时适配 W_q 和 W_v 效果最佳。** 即使秩仅为 4，也足以捕获 ΔW 中的足够信息，因此适配更多类型的权重矩阵比用更大的秩适配单一类型更优。

### 7.2 LoRA 的最优秩 r 是多少？

| 秩 r | W_q (WikiSQL) | W_q,W_v (WikiSQL) | W_q,W_v,W_k,W_o (WikiSQL) |
|------|--------------|-------------------|---------------------------|
| 1 | 68.8 | **73.4** | **74.1** |
| 2 | 69.6 | 73.3 | 73.7 |
| 4 | 70.5 | 73.7 | 74.0 |
| 8 | 70.4 | 73.8 | 74.0 |
| 64 | 70.0 | 73.5 | 73.9 |

**令人惊讶的发现：对于 {W_q, W_v}，秩 r=1 就已经足够了！** 这表明更新矩阵 ΔW 可能具有非常小的"内在秩"。

### 7.3 适配矩阵 ΔW 与原始权重 W 的关系

通过将 W 投影到 ΔW 的 r 维子空间进行分析：

**几个重要结论：**

1. **ΔW 与 W 有较强的相关性**（比随机矩阵强），说明 ΔW 放大了 W 中已有的某些特征。

2. **ΔW 不是简单重复 W 的顶部奇异方向**，而是放大了那些在 W 中**不被强调**的方向。

3. **放大因子非常大**：对于 r=4，放大因子约为 **21.5 倍**。

> 这表明，低秩适配矩阵**放大了那些在通用预训练中被学到但未被强调的、对特定下游任务重要的特征**。

## 8 结论与未来工作

微调巨大的语言模型在硬件需求和存储/切换成本方面代价高昂。我们提出了 LoRA，一种高效的适配策略，它**不引入推理延迟，不减少输入序列长度，同时保持高模型质量**。重要的是，通过共享绝大多数模型参数，LoRA 允许在部署为服务时快速切换任务。

**未来方向：**

1. LoRA 可以与其他高效适配方法结合，可能提供正交的改进。
2. 微调或 LoRA 背后的机制还远未清楚——预训练中学到的特征如何被转化以在下游任务上表现良好？LoRA 使得研究这个问题比全量微调更容易。
3. 我们主要依赖启发式方法来选择应用 LoRA 的权重矩阵，是否有更有原则的方法？
4. ΔW 的秩缺陷暗示 W 本身也可能是秩缺陷的，这也可以成为未来工作的灵感来源。
